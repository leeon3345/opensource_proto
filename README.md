# 국가사이버안보센터(NCSC) 가이드라인 준수 인프라 보안 자동 조치 플랫폼

---

## 1. 프로젝트 배경 및 목적

### 컴플라이언스 복잡성 증대

* 공공 클라우드 보안인증(CSAP) 및 NCSC 가이드라인의 방대한 요구사항 존재
* 체계적인 관리 필요

### 수동 조치의 한계

* 보안 위반 사항 대응 시 담당자의 수동 개입 필요
* 시간 소요 및 인적 오류 발생 위험 존재

### 자동화된 거버넌스 구현

* Policy as Code 기반 정책 관리
* AI 결합을 통한 실시간 진단 및 즉각적인 조치 프로세스 구축

---

## 2. 핵심 기능 명세

### ❶ 실시간 보안 대시보드 (Governance)

* 상태 요약 위젯

  * 위험 (Critical)
  * 경고 (Warning)
  * 정상 (Passed)
* 보안 점수 인디케이터

  * 정책 준수율 기반 실시간 건전성 점수 산출
* Glassmorphism UI

  * 반투명 카드 및 블러 기반 시각적 레이어

---

### ❷ 스마트 YAML 매니페스트 스캐너 (Detection)

* OPA 기반 정책 검증

  * Rego 언어로 작성된 NCSC 보안 규칙과 1:1 비교
* 스트리밍 로그 시뮬레이션

  * 터미널 UI 기반 실시간 분석 로그 출력
* 파일 파싱 및 정적 분석

  * YAML / JSON 설정 파일 내 취약 설정 탐지

---

### ❸ AI 기반 자동 조치 비서 (Remediation)

* 보안 강화 코드 제안

  * Bad Case vs Good Case 비교 생성
* 조항 매핑 해설

  * NCSC 가이드라인 조항 번호 표시
* Diff View 비교

  * 줄 번호
  * 문법 하이라이팅 적용

---

### ❹ 원클릭 보안 패치 및 적용 (Action)

* 코드 자동 업데이트

  * 버튼 클릭 기반 즉각적 반영
* 결과물 다운로드

  * 보안 패치 완료된 YAML 파일 제공
* GitOps 연동 시뮬레이션

  * 저장소 동기화 상태 표시

---

## 3. 시스템 아키텍처 및 데이터 흐름

### 데이터 처리 프로세스

1. 입력 단계

   * YAML 파일 업로드 및 데이터 추출

2. 진단 단계

   * OPA 엔진 기반 정책 위반 여부 판단

3. 분석 단계

   * 위반 데이터 → LLM(Gemini API) 전달
   * 해결 방안 생성

4. 조치 단계

   * 안전 코드 UI 반영
   * 원클릭 패치 실행

5. 사후 관리

   * 감사 로그 저장
   * 대시보드 수치 갱신

## 5. 구현 상세 전략

### 📂 정책 관리 전략

* Policy as Code

  * 모든 보안 규칙을 `.rego` 파일로 관리
  * 버전 관리 및 확장성 확보

* Mapping Data

  * 정책 ID ↔ NCSC 조항 매핑 JSON 구성

---

### 🤖 AI 프롬프트 엔지니어링

* Context Injection

  * OPA 결과 + NCSC 원문을 입력값으로 제공
  * 환각(Hallucination) 방지

* Output Consistency

  * 정해진 형식의 코드 출력 유도
  * 불필요한 텍스트 제거

---

### 🎨 UI/UX 고도화

* 몰입형 다크 모드

  * 붉은 네온 톤 + 블러 카드
* 가로 스크롤 캐러셀

  * 대시보드 → 스캐너 → AI 비서 흐름 연결
  * 자연스러운 화면 전환

---

* 운영 효율성 및 안정성 강화
